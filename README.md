## Таблица атрибутов и настроек JSON

| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт имя поля в JSON | `"name"` вместо `"Name"` |
| `[JsonIgnore]` | Поле не попадает в JSON | Пароли, внутренние заметки (`InternalNotes`) |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `"realName"` вместо `"RealName"` |
| `JsonStringEnumConverter` | Enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами | Для удобства при разработке |

## Главные выводы

### 1. Что такое сериализация и десериализация

**Сериализация** — это процесс преобразования объекта C#  в строку формата JSON.  
**Десериализация** — обратный процесс: из JSON-строки создаётся объект C#.

### 2. Зачем нужен `[JsonIgnore]` — реальный пример

`[JsonIgnore]` исключает свойство из сериализации, то есть оно не появится в JSON-ответе и не будет отправлено клиенту.

**Реальный пример:**  
В классе `Hero` есть поле `InternalNotes` — служебные заметки разработчика (например, «Любимый герой редактора», «Без суперсил, только деньги»). Эта информация не должна быть доступна пользователям API. Повесив `[JsonIgnore]`, мы гарантируем, что такие данные никогда не утекут наружу.

### 3. Почему enum лучше сериализовать строкой, а не числом

Если enum (`Universe`) сериализуется как число (`0` для Marvel, `1` для DC), клиент получает непонятное значение. Ему нужно либо знать соответствие чисел, либо выполнять лишний запрос для расшифровки.  


### 4. Что изменилось в ответе после настройки CamelCase

До настройки `PropertyNamingPolicy.CamelCase` имена полей в JSON соответствовали C#-свойствам:  
`"RealName"`, `"PowerLevel"`, `"Universe"` (с большой буквы).

После настройки все имена стали в **camelCase** (первая буква строчная):  
`"realName"`, `"powerLevel"`, `"universe"`.
